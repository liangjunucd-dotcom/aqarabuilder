# 20260403 Aqara Studio商业化路演 —— 会议纪要

未来，我们必须逐步走向 B2B 解决方案业务。因为客户真正购买的，不只是单个设备，而是面向不同场景的整体解决方案。无论是住宅、办公、楼宇、门店还是其他行业，场景需求都非常丰富，不可能全部由 Aqara 自己逐一完成。像民生银行、西门子这样的项目，对我们来说更重要的意义，是作为样板和打样，验证能力、沉淀方法、形成标杆，而不是长期依赖总部亲自去做每一个项目。

所以，我们接下来真正要做的，是把这些能力产品化、平台化，并通过 Builder 和社区对外开放。我们要让外部用户能够清楚理解 Aqara Studio 到底是什么、能接入哪些协议、能解决哪些问题、如何上手、如何扩展。无论对方是普通用户、集成商、开发者，还是潜在合作伙伴，都应该能在平台上看到适合自己的介绍内容，包括产品能力说明、解决方案案例、演示视频、接入方式和实践路径。用户既可以先在云端体验，也可以购买一台 M300 自己动手尝试。开发者如果有自己的硬件设备，也应该能清楚知道如何接入 Studio，接入之后又能基于 Aqara 的体系去创造新的玩法和业务机会。

更重要的是，这套体系应该尽可能让他们自己完成探索和成长，而不是凡事都依赖绿米工程师介入。比如，一个做楼宇项目的人想做节能方案，他不一定要先找到总部支持，而是可以直接在社区里查看资料、发帖提问、找到做过类似项目的人，甚至直接找到可以合作的服务方。也就是说，我们要把原本依赖内部交付的能力，逐步变成一个由平台、社区和合作网络共同支撑的外部供给体系。

在这个过程中，很多人一开始可能只是玩家、开发者或集成商，但在不断尝试和分享的过程中，他们会发现自己不仅能用 Aqara 做项目，还能通过帮助别人解决问题获得收益。这样，他们就会自然成长为 Installer，成为我们的服务网络的一部分。

未来，这一切都应该是线上化管理的。无论是有人希望成为 Installer，还是想申请开设 Aqara Space 门店，我们都应该有一套线上账户、审核、认证和分级体系。平台可以为他们建立账户，对其进行资格审核和能力认证，通过培训和测试后，再赋予不同等级的权限。认证后的角色可以在线查看对应的资料、价格体系、收益空间、解决方案包和服务政策，也能够更清楚地知道，自己卖一套方案、交付一个项目，能获得怎样的利润和回报。

与此同时，平台本身也要成为一个强大的管理和服务入口。不同角色进入平台后，都能被清晰引导到自己所需的资料、能力和路径上；平台内也应该有 Agent，能够帮助他们查询资料、理解规则、生成方案、辅助创作和完成业务动作。这样，平台不只是一个网站，而是一个真正帮助每类用户成长和成交的线上 operating system。

当越来越多的人在平台上创作、分享和沉淀案例后，这些内容本身也会成为业务增长的入口。有人为了吸引客户，会主动分享自己的方案、案例和经验；而陌生用户来到平台后，就会直观看到 Aqara 能解决如此多样的实际问题。久而久之，平台会自然形成按行业、按场景、按地区、按能力分类的内容与服务网络。用户不仅能看到各种案例，还能看到“谁在这个行业做得好”“谁在这个地区有经验”“谁是被认证的服务提供者”。如果分享者本身是注册且认证过的 Installer 或服务商，用户甚至可以直接联系对方，委托其承接项目。

对于 B2C 用户，我们依然需要一个清晰的产品站逻辑。因为在 B2C 场景下，买的人和用的人通常是同一个人，他们需要的是看得懂、买得明白、买完能用好。所以平台也必须同时具备优秀的消费者产品展示和购买体验，让用户在充分了解产品、场景和价值之后，再做决策。

而从更长期来看，我们还要在全球逐步建立 Aqara Space 门店网络。每一个 Aqara Space，不只是一个卖货点，更应该是当地的展示、服务、培训和交付节点。对于附近的 Installer 或用户来说，Aqara Space 所拥有的资源、能力和支持，都应该能够成为他们可调用的一部分。这样，Aqara Space 就不只是线下门店，而是我们在每个地区的本地服务基础设施。
