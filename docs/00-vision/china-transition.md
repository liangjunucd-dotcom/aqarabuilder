# China Transition: From 服务商 to 区域运营伙伴

> 中国市场存量服务商体系是宝贵资产，但向"平台直管 Builder Network"演进时存在结构性利益冲突。
> 本文档定义化解机制：**服务商 → 区域运营伙伴**的产品化、参数化转型路径。

---

## 一、为什么必须转型

### 服务商既是资产也是瓶颈

| 资产维度 | 瓶颈维度 |
|---|---|
| 中国深度渠道、客户信任 | **掌控 Installer 限制人才流动**（人才无法跨城市 / 跨服务商发展） |
| 体验中心、前置仓、本地售后能力 | **培训质量参差**（不同门店标准不一） |
| 区域客户关系积累 | **海外市场无法直接复制服务商体系** |
| 销售能力、对外报价能力 | **与平台直管诉求结构性冲突**（谁拥有 Builder） |

### 长期方向

> Aqara 必须建立 **平台直管的 Builder Network**——这是与 Apple Home / Google Home / Matter 的根本差异，也是全球扩张的前提。详见 [`global-network-model.md`](./global-network-model.md)。

> 但中国服务商体系是宝贵资产，**不能"砍掉"**，必须**产品化转型**。

---

## 二、核心原则

> **不强制过渡。通过让"打开开关 = 赚更多钱"的激励成立，让服务商自愿转型为区域运营伙伴。**

### 关键概念变化

| 旧定位 | 新定位 |
|---|---|
| 渠道分销商 + Installer 雇主 | **区域运营伙伴** = 本地基础设施提供者 |
| 控制 Installer，靠工程项目分成 | 提供前置仓 + 体验中心 + 招募，靠**区域被动分成** |
| 培训 Installer | **不再培训**——培训彻底剥离到 Aqara Academy |
| Installer 是雇员 | Builder 是 Aqara 平台直管，区域伙伴是 **Affiliation 选项之一** |

详见 [`../02-roles/role-model.md`](../02-roles/role-model.md) 和 [`global-network-model.md`](./global-network-model.md)。

## 三、四层开关（区域伙伴后台可配置）

| # | 开关 | 默认 | 说明 |
|---|---|---|---|
| 1 | **区域单优先权** | 开 | 新单先给区域伙伴关联 Builder，超时（如 30 min）才外放 |
| 2 | **外部 Builder 准入** | 关 | 是否允许外部 Builder 接区域订单。打开后享受被动分成 |
| 3 | **Builder 挂靠邀请** | 开 | 允许独立 Builder 挂靠到自己名下，分挂靠抽成 |
| 4 | **订阅分佣区域归属** | 开 | 自动按规则分配区域内订阅分佣 |

> 开关在区域伙伴后台呈现为"控制面板 + 被动收入预估"——这是和服务商谈转型的核心沟通工具。

---

## 四、五类被动收入（"不做单也能赚"）

| 收入项 | 倾向比例 | 来源 |
|---|---|---|
| 区域订阅分成 | 10–15% | 区域内所有 Aqara AI 订阅 |
| 挂靠 Builder 订单抽成 | ~10% | 挂靠 Builder 接的所有订单 |
| 前置仓发货分成 | 3–5% | 区域内硬件履约 |
| Studio 部署激励 | 按台数 | 每部署一台 Studio 给区域伙伴的激励 |
| 招募补贴 | 按人头 | Aqara 按区域贡献的认证 Builder 数量补贴 |

> 区域伙伴**不再承担培训**——培训由 Aqara Academy 直接交付。"招募补贴"是因为伙伴提供场地 / 招募协助，**不是因为他们培训**。

## 五、三阶段开放节奏（中国市场）

| 阶段 | 时间 | 策略 |
|---|---|---|
| 阶段 1 — 保护期 | 2026 – 2027 | 默认保守配置，培养挂靠 + 被动分成的甜头；样板服务商先行 |
| 阶段 2 — 引导期 | 2027 – 2028 | 区域开放奖励计划；个别城市试点；样板效应辐射 |
| 阶段 3 — 全面开放 | 2028+ | 新城市直接全开；老城市保留优先权但不排他 |

详细里程碑见 [`../04-roadmap/milestones.md`](../04-roadmap/milestones.md)。

## 六、海外市场（无过渡）

> **海外没有服务商包袱，直接跳到阶段 3 等价配置——纯直管 Builder 网络。**

- **灯塔市场（阿联酋）**：Phase 1 验证模型，无区域伙伴层，City Lead 制
- **高端市场（美 / 欧 / 日）**：Phase 2 复制阿联酋经验，City Lead 制
- **中等 / 新兴市场**：Phase 3 用 **Master Franchise** 模式（区域代理 = 区域运营伙伴），但 Builder 仍由 Aqara 平台直管

> **同一平台，参数化配置适配不同市场/阶段，不做代码分叉。**

中国头部服务商可作为 **海外 Master Franchise 输出方**——给他们一个新增长曲线，化解抵触情绪。

详见 [`global-network-model.md`](./global-network-model.md)。

## 七、产品上的体现

| 模块 | 设计要点 |
|---|---|
| 区域伙伴后台 | 控制面板 + 被动收入预估 + Builder 招募管理 + 物流管理 + 体验中心管理 |
| 数据模型 | `affiliation` 字段表达 Builder 与区域伙伴关系；可变，Badge 跟人走 |
| 分佣引擎 | 支持 **区域归属 + 挂靠归属 + 执行归属** 三维度，规则参数化 |
| Lead 派单 | 按开关 #1 决定派单优先级 |
| 培训入口 | 永远指向 Aqara Academy，区域伙伴 UI **不出现"培训管理"**，只出现"招募协助 / 场地管理" |

---

## 八、落地原则

1. 任何"服务商 vs Builder"相关的功能设计都要考虑这个开关架构。
2. 分佣规则实现必须支持 **区域归属 + 挂靠归属 + 执行归属** 三维度。
3. 区域伙伴后台的数据模型要支持区域策略配置。
4. 和服务商谈转型时，"开关控制面板 + 被动收入预估 + Studio 部署激励"是核心沟通工具。
5. 产品形态上，用策略参数配置适配不同市场/阶段，**不做代码分叉**。
6. 区域伙伴 UI 中**不出现"培训管理"**——培训永远指向 Aqara Academy。
7. **Builder 个人数据 / Badge / 评价跟人走**，区域伙伴变更不影响 Builder 资产。
