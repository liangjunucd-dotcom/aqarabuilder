## 一、需求背景

Aqara 现有用户端 App（Aqara Home）是以设备管理为中心的通用型 App，所有家庭成员共享同一套界面，功能以固定菜单形式存在。随着 Aqara 战略向「专业 Installer 交付智能空间」方向演进，现有 App 存在以下核心问题：

1. **无法体现 Installer 的交付价值**：Installer 辛苦配置好的自动化、场景和服务，在 App 里与用户自己添加的设备混在一起，毫无品质感，Installer 的专业性无处呈现；
2. **一家人用同一套界面**：主人、老人、儿童、保姆对智能家居的需求截然不同，强迫他们用同一套 UI 是典型的「为了智能而智能」；
3. **功能固化、无法扩展**：新功能要发版才能上线，用户无法按需取用，也无法支持第三方生态接入；
4. **与 Studio 中枢的关系未打通**：App 与本地 Studio 之间的关系不清晰，用户无法直观理解本地与云端的边界。

为此，需要构建全新的用户端 App —— **Aqara Life**，以 Shell + Plugin 架构为基础，以千人千面的个性化交付为核心差异点，让 Installer 的专业价值得以具现化，同时为普通用户提供开箱即用的智能家居体验。

---

## 二、产品定位

**Aqara Life** 是 Aqara 面向终端用户的全新移动端 App，定位为「智能家居个人界面容器」。App 本体（Shell）只承担账号体系、Studio 发现与绑定、Plugin 运行沙箱、AI 入口等基础职能，所有功能由**插件（Plugin）**填充。

目标用户：持有或即将购买 Aqara Studio 的家庭用户，以及通过 Installer 获得 Aqara 全屋方案的终端客户。

核心价值主张：

- **对普通用户**：下载即用，Installer 已经配置好了你的专属界面，开箱就是高级感；
- **对 Installer**：交付物不再是一堆设备，而是一套有名字、有品牌感的个性化 App 配置；
- **对 Aqara**：App 生态化，第三方插件带来持续的平台收益和用户黏性。

## 三、设计原则


| 序号  | 原则                 | 说明                                                                           |
| --- | ------------------ | ---------------------------------------------------------------------------- |
| 1   | **Shell 不承载业务逻辑**  | App 主体只提供导航框架和账号体系，功能全部由 Plugin 提供。避免 App 主体膨胀。                              |
| 2   | **千人千面，成员隔离**      | 同一家庭账号下，每位成员（主人/老人/儿童/保姆）看到各自独立配置的首页，互不干扰。                                   |
| 3   | **三层配置，向下兼容**      | 配置优先级：用户微调 > Installer 叠加 > Aqara 默认。升级时保留 Installer 层，用户可一键恢复 Installer 配置。 |
| 4   | **本地优先**           | 核心数据和自动化运行在 Studio 本地，云端只提供远程代理隧道和增值服务，不托管用户数据。                              |
| 5   | **Installer 价值可见** | 首次激活 Installer 配置时有「揭盒」仪式感；服务卡片标注来源；用户可看到谁为自己配置了什么。                          |
| 6   | **渐进解锁**           | 未绑定 Studio 时 App 可用；绑定后解锁更多功能；Installer 交付后界面升级为专属配置版本。                      |

## 四、功能需求

### 4.1 登录与注册

**功能概述**：用户通过 Aqara 账号登录 Aqara Life，登录成功后进入 App 主界面。

**交互流程**：

1. 冷启动 App，显示启动页（Logo + 品牌名 + 加载文案「正在加载你的家…」），动画时长约 2 秒后跳转
2. 未登录用户落地「登录」页面，展示：地区选择（🇨🇳 中国大陆 / 🇸🇬 新加坡 / 🇺🇸 美国 / 🇪🇺 欧洲）、手机号/邮箱输入、密码输入
3. 点击「登录」→ 校验通过 → 播放首页加载动画 → 进入主界面
4. 支持「注册账号」跳转注册流程（本 PRD 不展开）
5. 支持「忘记密码」跳转找回流程（本 PRD 不展开）

**界面状态**：


| 状态    | 登录按钮     | 说明         |
| ----- | -------- | ---------- |
| 输入不完整 | 灰色，不可点击  | 手机号或密码为空时  |
| 输入完整  | 主色填充，可点击 | —          |
| 登录中   | 加载态      | 显示 spinner |
| 登录失败  | 恢复可点击    | 行内展示错误提示   |


**地区选择**：地区影响账号所在服务器，不同地区账号相互隔离（不通用）。地区选择器以底部弹出 Sheet 形式呈现，列表带国旗 Emoji + 地区名，支持点击选中。

---

### 4.2 首页（Home）

首页是 Aqara Life 的核心界面，根据用户的 **Studio 绑定状态** 和 **Installer 配置状态** 动态渲染为三种模式：

#### 4.2.1 Solo 模式（无 Studio）

**触发条件**：用户未绑定任何 Studio

**界面构成**：

- 顶栏：「Home」标题 + AI 入口按钮 + 通知铃铛
- AI 辅助栏：快捷提问入口（见 4.6）
- Studio 发现横幅：若局域网内发现未绑定 Studio，在 AI 栏下方显示「发现附近 1 台未绑定 Studio」引导条，带动态绿点；右侧 X 按钮可关闭
- 插件市场区块（见 4.5）
- 底部导航栏：「发现 / 设备 / 我的」三 Tab

> 💡 设计意图：Solo 模式下首页主要功能是引导用户绑定 Studio 或浏览插件市场，不强迫用户立即操作，符合「渐进解锁」原则。

#### 4.2.2 Studio 模式（已绑定 Studio，无 Installer 配置）

**触发条件**：用户绑定了 Studio（URL 参数 `mode=remote`），但尚未接受 Installer 配置

**界面构成**：

- 顶栏：「Home」标题 + **Studio 小标签**（Studio 名称，绿色背景）+ AI 入口 + 通知铃铛
- AI 辅助栏
- 「常用」区块：展示空状态引导文案「配置你的第一套方案」，下方向下箭头呼吸动画引导探索
- 插件市场区块
- 底部导航栏：同 Solo 模式（「发现 / 设备 / 我的」）

#### 4.2.3 Installer 配置模式（已接受 Installer 专属配置）

**触发条件**：用户接受了 Installer 推送的个性化配置（URL 参数 `config=accepted&persona=<角色>`）

**界面构成**：

- 顶栏：「Home」标题 + Studio 标签 + AI 入口 + 通知铃铛
- AI 辅助栏
- **服务区块**：Installer 配置的服务卡片列表（最多 3 条），每张卡片含图标/名称/状态/描述，点击进入服务详情（见 4.9）
- **场景区块**（仅 dad 角色）：横向滑动场景胶囊列表，支持点击激活（高亮态）
- **设备区块**（仅 dad 角色）：2 列设备卡片，每张含图标/名称/位置/状态/开关按钮
- 精简插件市场（`compact` 模式）
- **专属 Tab 栏**：根据 Persona 展示不同 Tab 组合（见下表），浮动圆角胶囊样式

**Persona Tab 配置**：


| Persona     | Tab 列表             |
| ----------- | ------------------ |
| dad（主人）     | 首页 · 设备 · 场景 · 我的  |
| kid（儿童）     | 首页 · 玩法 · 我的       |
| nanny（保姆）   | 首页 · 任务 · 我的       |
| elderly（老人） | 首页 · 设备 · 我的（大字模式） |


#### 4.2.4 Installer 配置申请 Sheet

**触发条件**：用户进入带有 `config=pending` 参数的 Studio（即 Installer 推送了配置但用户尚未接受）

**交互**：

1. 进入首页时，底部弹出 Sheet，覆盖在普通首页之上
2. Sheet 展示：Installer 名称、Persona 角色预览（Tab 列表以胶囊形式展示）、接受/拒绝按钮
3. 点击「接受配置」→ 播放**揭盒动画**（见 4.2.5）→ 动画结束后跳转为 Installer 配置模式首页
4. 点击「暂不需要」→ 关闭 Sheet，以 Studio 模式（无 Installer 配置）继续使用

#### 4.2.5 揭盒动画（Apply Reveal）

**触发时机**：用户接受 Installer 配置时、或在插件详情页点击「应用配置」时

**动画时序**：

1. 阶段一（0–700ms）：全屏覆盖，中心出现抖动📦 + 文案「正在应用配置…」
2. 阶段二（700–1200ms）：📦 消失，渐变色光圈爆炸扩散 + 彩色粒子飞散（6 个方向）+ 🎁 出现
3. 阶段三（1200–2800ms）：全屏展示 Persona 图标（大尺寸）+ Persona 名称 + 「配置已激活 🎉」+ 三点呼吸动画
4. 2800ms 后自动结束，跳转目标页

**Persona 视觉配置**：


| Persona | 图标  | 名称   | 渐变色  |
| ------- | --- | ---- | ---- |
| dad     | 🏡  | 主人模式 | 蓝-靛蓝 |
| kid     | 🌈  | 儿童模式 | 橙-琥珀 |
| nanny   | 📋  | 管家模式 | 青-靛青 |
| elderly | 🌿  | 关爱模式 | 绿-翠绿 |

### 4.3 Studio 管理

#### 4.3.1 Studio 列表

**功能概述**：展示用户账号下所有 Studio，区分在线状态和连接方式。

**入口**：「我的」Tab → Studio 区块；或顶栏 Studio 标签点击

**列表项构成**：

- M300 硬件示意图标（小型 SVG，含 LED 状态指示灯）
- Studio 名称
- 状态 Badge：在线（绿点）/ 局域网（紫点）/ 离线（灰点）
- 连接方式图标：`cloud`（手机→互联网→Studio）/ `local`（手机→WiFi→Studio）
- 是否有待处理的 Installer 配置（有则进入时弹确认 Sheet）

**连接方式说明**：


| 类型      | 含义                      | 入口行为                         |
| ------- | ----------------------- | ---------------------------- |
| cloud   | 通过 Builder Cloud 远程隧道访问 | 直接连接，若有 Installer 配置先弹 Sheet |
| local   | 局域网直连，尚未绑定 Builder      | 先走「连接」动画页，再进首页               |
| offline | 离线不可访问                  | 点击无反应（或 Toast 提示）            |


**底部操作**：「+ 添加 Studio」按钮

#### 4.3.2 添加 Studio 流程

**步骤 1 — 局域网扫描**

1. 进入「添加 Studio」页，立即开始扫描局域网（mDNS）
2. 扫描中：中心区域 Wi-Fi 图标 + 多圈脉冲扩散动画 + 文案「正在扫描局域网…」+ 当前网络名
3. 扫描完成（约 2s）：发现的 Studio 列表动画入场，展示「设备名 · IP 地址 · 设备 ID」
4. 若未发现设备：空状态 + 提示「确保 Studio 和手机在同一 Wi-Fi 下」
5. 点击「重新扫描」可重新触发扫描动画
6. 点击某台设备 → 进入步骤 2

**其他入口（非扫描）**：

- 扫描二维码：跳转系统相机/扫码页（本 PRD 不展开）
- 手动输入 IP/设备 ID：弹出输入框（本 PRD 不展开）

**步骤 2 — 初始化 Studio（Set Password + Bind Workspace）**

进入「初始化 Studio」页，顶部双步骤进度条。

**子步骤 1：设置本地账号密码**


| 字段   | 说明                      |
| ---- | ----------------------- |
| 用户名  | 固定 `super`，只读展示，带「默认」标签 |
| 密码   | 可见/隐藏切换；输入后展示密码规则验证列表   |
| 确认密码 | 与密码一致时右侧显示绿色勾           |


密码规则（全部满足才可下一步）：

- 8–16 位字符
- 包含大写、小写、数字或符号中至少两种
- 两次密码一致

**子步骤 2：绑定 Workspace**

- 展示用户 Builder 账号下的 Workspace 列表（按账号地区筛选同 DC 的 Workspace）
- 默认选中第一个；下拉展开选择器
- Workspace 卡片展示：彩色头像 + 名称 + Role + 数据中心（DC Badge）
- 安全提示块：「数据保留在本地 Studio，Builder 仅提供安全的远程访问隧道」
- 「连接到 Aqara Builder」主按钮 → 进入步骤 3
- 「暂时跳过」次级按钮 → 直接进入局域网连接动画（本地模式）

**步骤 3 — 绑定进度（Binding Progress）**

三步骤进度列表，依次推进：


| 步骤          | 图标          | 说明                               |
| ----------- | ----------- | -------------------------------- |
| 建立本地连接      | WiFi        | 与 Studio 局域网握手                   |
| 传递账号信息      | Send        | 将账号凭证交给 Studio                   |
| Studio 连接云端 | CloudUpload | Studio 正在注册到 Aqara Builder Cloud |


每步完成后对应行变为绿色 ✓，当前步骤图标旋转动画。全部完成后跳转至成功页：CheckCircle + 「连接成功」+ 「进入 Studio」按钮。

#### 4.3.3 连接动画页

**触发**：点击 local 类型 Studio 时，或绑定流程「暂时跳过」后

**展示**：设备图标 + 脉冲圆圈动画 + Studio 名称 + 三点闪烁「连接中」→ 约 1.6s 后变为「已连接」绿字，再 0.7s 后自动跳转首页

**连接路径图示**：手机图标 → 点线 → Studio 图标（local 模式省略中间的 Globe 图标）

#### 4.3.4 安装验收（Accept）

**触发**：Installer 完成安装后发送验收通知，用户点击通知进入

**页面构成**：

- Installer 信息卡（头像 + 姓名 + 公司 + 「已完成安装」Badge）
- 安装时间 + 安装地址
- 安装清单（可折叠，展示具体完成项 + CheckCircle）
- 星级评分（1–5 星，悬停高亮，选中后显示文字评价）
- 问题上报入口（折叠，选项：设备无响应/自动化未生效/部分设备未配对/其他问题）；选中问题后显示「48 小时内跟进处理」提示
- 「确认验收」主按钮（若有评分则显示「并提交评价」）

**验收完成**：

- 若已连接 Studio：直接显示成功页 + 「返回首页」按钮
- 若未连接 Studio：显示成功页 + 自动跳转连接动画

### 4.4 插件市场

**功能概述**：用户在首页下方的嵌入式插件市场区块浏览、筛选、安装插件。

**布局**：

- 分类筛选横向滑动胶囊（全部 / 安全服务 / 节能管理 / 健康生活 / 儿童模式 / 氛围照明）
- 「热门玩法」横向卡片列表：每张卡片含渐变背景/名称/Emoji/评分/安装量/「需要 Studio」角标
- 「插件列表」竖向卡片：支持按分类筛选

**插件卡片内容（列表态）**：

- 渐变色背景方形图标 + Emoji
- 插件名 + 作者标签（官方/ACB/开发者）
- 评分（点赞率 %）+ 安装量
- 价格（Free / ¥X.X）
- 若 `studioOnly: true` 且未绑定 Studio：「需要 Studio」角标，安装按钮置灰

**compact 模式**：Installer 配置首页下方的插件市场，仅展示分类胶囊 + 竖向卡片，无热门玩法大卡

#### 4.4.1 插件详情页

**入口**：点击任意插件卡片

**页面构成**：

- 顶部 Hero：全宽渐变背景 + 大尺寸 Emoji + 插件名 + 标签徽章（官方插件/ACB 认证/开发者）
- 功能描述正文
- 安装/已安装按钮区（见下方状态表）
- 若来自 Installer 推送（`installer` 参数存在）：展示「Installer 推荐」说明 + 安装后触发揭盒动画

**安装按钮状态**：


| 状态             | 按钮文案           | 按钮样式     |
| -------------- | -------------- | -------- |
| 未安装，可安装        | 安装             | 主色填充     |
| 下载中（0–100%）    | 进度条 + 暂停按钮     | 进度条样式    |
| 已暂停            | 继续 + 取消        | 两按钮      |
| 已安装            | 已安装            | 灰色不可点    |
| 需要 Studio（未绑定） | 「需要先添加 Studio」 | 警告样式，不可点 |


**进度条交互**：点击「暂停」→ 进度冻结，显示「继续」+「取消」；点击「继续」→ 进度从断点继续；点击「取消」→ 重置为未安装态

**安装后（来自 Installer 推送）**：点击「应用」→ 触发揭盒动画 → 动画结束后跳转 Installer 配置首页，同时在「我的」页将对应的 Pending Config 卡片标为已处理

### 4.5 AI 助手

**功能概述**：内嵌 AI 对话界面，支持快捷指令和自由对话，用于场景建议、设备查询、节能分析等。

#### 4.5.1 AI 辅助栏（首页嵌入）

出现在首页顶部 AI 区块，是进入 AI 对话页的快捷入口，展示样式类似输入框，点击跳转完整 AI 页面。

#### 4.5.2 AI 对话页

**入口**：AI 辅助栏点击 / 顶栏 AI 图标

**页面构成**：

- 顶栏：返回按钮 + Aqara AI 标题 + 渐变色 Logo 球 + 历史列表按钮
- 空状态：居中大尺寸 AI Logo + 快捷提问卡片 4 张（带 Emoji + 文案）
- 消息列表：用户消息（右对齐，主色背景）/ AI 消息（左对齐，深色卡片）
- 底部输入区：快捷 Chip 横向滑动（AI 调光/推荐玩法/节能建议/场景推荐/安防检查）+ 文本输入框 + 发送/语音按钮

**快捷 Chip 默认文案**：AI 调光 · 推荐玩法 · 节能建议 · 场景推荐 · 安防检查

**交互流程**：

1. 点击快捷 Chip 或输入自定义内容 → 发送后用户消息气泡入场
2. AI 侧出现「思考中」动画（三点脉冲）约 1.2s
3. AI 回复消息气泡入场，文案根据问题类型呈现（列表/普通文字）
4. 对话历史可滚动查看，自动滚动到最新消息

### 4.6 通知中心

**功能概述**：集中展示系统通知、Installer 推送、能耗报告、新插件上线等消息。

**入口**：首页顶栏铃铛图标

**通知类型与交互**：


| 类型                 | 示例                | 主操作          | 次操作 |
| ------------------ | ----------------- | ------------ | --- |
| `installer-plugin` | Installer 分享了专属配置 | 获取插件（→ 插件详情） | —   |

## 五、数据模型

### 5.1 核心实体关系

```
Account (Aqara 账号)
  └─ FamilyHome (家)
       ├─ Member[] (家庭成员)
       │    └─ Persona (角色配置：dad/kid/nanny/elderly)
       │         └─ InstallerConfig (Installer 交付的配置层)
       └─ Studio[] (绑定的 Studio 设备)
            ├─ accessType: cloud | local | offline
            └─ workspace: Workspace (绑定的 Builder Workspace)

Plugin (插件)
  ├─ id, name, category, tag (Official/ACB/Developer)
  ├─ studioOnly: boolean
  └─ price: Free | ¥X.X

InstallerConfig
  ├─ persona: Persona
  ├─ services: Service[]
  ├─ scenes?: Scene[]
  └─ devices?: DeviceGroup[]
```

### 5.2 三层配置优先级

```
用户微调层 (Layer 3) — 优先级最高
    ↑ 覆盖
Installer 配置层 (Layer 2) — Installer 交付
    ↑ 覆盖
Aqara 默认层 (Layer 1) — 基于 Persona Role 的默认值
```

## 六、非功能需求

1. **启动耗时**：冷启动到首页可交互 ≤ 3s（包含动画时长）
2. **连接建立**：局域网连接建立 ≤ 2s；云端隧道连接 ≤ 5s
3. **Studio 离线时**：App 仍可浏览插件市场和「我的」页，不崩溃，不白屏；离线 Studio 展示灰色 LED 状态
4. **动画帧率**：关键动画（揭盒、连接进度）保持 60fps
5. **安全**：Studio 本地密码加密传输；Builder 连接使用 TLS；Local 账号凭证不明文存储
6. **多端一致性**：同一 Account 在不同手机上登录，Persona 配置、Studio 绑定状态保持一致

1. **Q：家庭成员账号切换的具体交互是什么？** Persona 切换是同一账号下的界面模式切换，还是需要多账号登出/登入？影响：用户体验设计、账号模型实现复杂度。

## 附录：术语表


| 术语                         | 释义                                                           |
| -------------------------- | ------------------------------------------------------------ |
| **Studio**                 | Aqara 智能家居中枢硬件设备（如 M300），承载本地自动化和设备网关功能                      |
| **Plugin（插件）**             | Aqara Life App 中的功能单元，运行在 Shell 框架内                          |

| **Workspace**              | Builder 平台的项目空间，每个 Workspace 对应一套 Studio Cloud 后台            |
| **Installer**              | 专业安装/配置服务商，通过 Builder 平台管理项目和交付配置                            |
| **ACB**                    | Aqara Certified Builder，Aqara 认证的 Builder 开发者，其发布的插件带 ACB 标签 |
| **三层配置（Cascading Config）** | Aqara 默认 → Installer 叠加 → 用户微调 的配置继承机制                       |
| **灵感值**                    | AI 管家订阅的使用额度单位，每次 AI 能力调用消耗一定数量                              |
| **揭盒动画（Apply Reveal）**     | Installer 配置激活时的仪式感全屏动画，强化交付价值感知                             |
| **Edge2Cloud**             | Studio 本地设备通过 Aqara Builder Cloud 建立安全远程隧道的连接机制              |
| **mDNS**                   | 局域网设备发现协议，App 通过它自动发现同网络内的 Studio                            |
