# Builder Workshop:AI 原生空间智能创作平台

> **AI 编剧 + 导演,一人 Smart Home 工作室** —— ACB / Professional / Customer 在统一平台上完成"创作 → 沙箱 → 发布 → Apply → 持续服务"全周期。
> **项目是一等公民**:一份方案 = 一个项目,可批量部署到 N 台 Studio(自家 / 授权客户家)。
> **身份渐进 + AI 算力档 二维正交**:同一套 UI 按身份渐进解锁,AI 算力档按任务自由切换。
> **入口边界**:Workshop 是创作能力集合，不是 Builder Pro 默认一级菜单；售前方案进入 `Projects -> Design`，插件 / 服务包供给进入 `Company -> Catalog`。
> **交付后持续服务是 Aqara 独有护城河** —— 项目永不结束,Studio 物理锚点 + 数据回流 + AI 预警 + 订阅续约。

---

## 术语备注

| 术语 | 含义 |
|---|---|
| **Workshop** | Builder Pro 内的创作能力集合；不作为默认一级菜单 |
| **Forge / Builder** | Workshop 内的具体创作或开发工具。售前 Space Design、已接入 Studio 后的 Life Dashboard、开发者 Plugin Builder 按生命周期分流 |
| **项目(Project)** | 一份业务上下文。售前项目沉淀 DesignPlan；已交付家庭沉淀 Home Snapshot / PersonaDashboard；插件项目沉淀 PluginPackage |
| **作品(Creation)** | 项目的对外展示形态,在本文等同"项目" |
| **空间图谱(Spatial Graph)** | Space Design 的核心产物,用于方案、报价、施工与后续部署映射 |

⚠ 与 Aqara Space 门店、Space 数据模型的术语区分见 [`../03-architecture/service-billing-model.md`](../03-architecture/service-billing-model.md) 顶部"术语备注"。

## 一、核心心智

### 1.1 AI 漫剧式的"导演工坊"

借剧火(juhuo.cn)"AI 编剧 + 导演,一人短剧工作室"心智 —— Workshop 让每个 ACB 不再是"某品牌的认证工程师",而是 **"为这家人的智能家居作品的导演"**。

| AI 漫剧 | Aqara Workshop |
|---|---|
| 短剧项目 | 客户项目(一份方案) |
| AI 编剧助手 | AI 方案助手 |
| 画风预设(美型古装 / 张艺谋风格) | **空间风格主题**(关爱老人 / 科技极客 / 极简主义 / 亲子家庭 / 奢华度假) |
| 基础设定(集模式 / 比例 / 时长) | **基础设定**(户型 / 家庭成员 / 预算 / 智能化深度 / 文化偏好) |
| 大纲与剧本 | 空间方案 + 自动化策略 |
| 角色 | **家庭成员 + Persona**(主人 / 老人 / 孩子 / 宠物 / 访客) |
| 场景 | 房间 + 场景模式(晚安 / 出门 / 欢聚 / 深夜警戒) |
| 道具 | 设备清单 + SKU |
| 分镜 | 自动化时间线 / 触发剧情 |
| 项目画板 | Workshop 画板(户型 3D + Forge 工具集) |
| AI 算力档(总设计师 / 方案师 / 草图师) | **AI 算力档**(总设计师 5x / 方案师 3x / 草图师 0.5x) |

> "导演"心智不仅降低门槛,也是 ACB 内容飞轮的情感基础。

### 1.2 项目 = 一等公民,自用 / 客户不区分

ACB 日常做的不是"用某个 Forge",而是 **"为某个项目做事"**。一个项目可以是:
- 我自己家的 Smart Home 方案
- 我为李先生家做的方案
- 我做的标配模板,可以批量卖给 N 个家庭

**关键设计**:项目本身不区分"自用 vs 客户",只有两个正交维度:

| 维度 | 含义 |
|---|---|
| **部署授权(deployment_grants)** | 我有哪些 Studio 的部署权 — 自家 Studio + 客户签过授权的 Studio |
| **可见性(visibility)** | Draft / Private / Published / Verified / Commercial 五档 |

这把"作品创作"和"客户交付"解耦 — 同一项目可以只给自己用,也可以变成商品方案批量部署到 N 个客户家。

### 1.3 持续服务 — 项目永不结束

Aqara 真正的差异化护城河:**装修 / 家电 / 智能家居平台做完项目就拜拜,Aqara 的 Studio 物理锚点 + 数据回流 = 项目永远在线**。

ACB 在 Workshop 里的客户项目永远有 [持续服务] tab:
- 🟢 Studio 健康监控(CPU / 内存 / 在线率)
- ⚠ AI 预警(网关走高 / 传感器灵敏度降低 / 自动化频次异常)
- 📊 真实使用数据(自动化触发次数 / 用户手动调整 / 客户反馈)
- 🔄 推荐下次服务动作
- 📅 订阅续约管理

ACB 从"一次性装修工" → **"家庭智能家居家庭医生"**。

## 二、身份 × AI 算力档 二维模型

### 2.1 身份渐进(同一套 UI)

```
普通用户 (Customer, 必有)
    ↓ 自助进阶,无门槛
Professional (自助进入专业工作台,部分能力受限)
    ↓ 经 Aqara Academy 认证 + Badge
ACB (Aqara Certified Builder, 全开通)
    └── 子角色可叠加: ACB(Installer/Designer/Developer/Service)
```

**核心设计**:**不做 4 套 UI**,同一套 UI 按身份渐进解锁。借 Notion / Figma / Linear 的 workspace 模式 —— 按角色显示不同面板,但底层是同一个平台。

### 2.2 AI 算力档 — 跟身份正交

剧火启发:AI 编剧分顶级 / 专业 / 助理三档,这不是身份,是**按任务选 AI 能力**(类似 GPT-5 / GPT-5-mini)。Aqara Workshop 采用**空间设计职业层级**命名:

| 算力档 | 算力消耗 | 适合场景 |
|---|---|---|
| **总设计师**(5x) | 高 | 从零新项目 / 大户型复杂方案 / 全屋 Persona 重做 |
| **方案师**(3x) | 中 | 已有方案优化 / 标配方案微调 / 风格改写 |
| **草图师**(0.5x) | 低 | 格式转换 / 单点调整 / 渲染预览 |

**关键**:跟用户身份彻底解耦。新手也能召唤总设计师做精品,ACB 也可以图快用草图师。每次会话/任务都可切换。

### 2.3 四档能力 × 同一套 UI

| 能力档 | 解锁条件 | 能做什么 | 对标 |
|---|---|---|---|
| **L1 浏览 + Apply** | Customer 默认 | 看别人作品 + 一键 Apply 到自家 Studio;不编辑 | Spotify / Netflix 消费者 |
| **L2 轻量创作** | Customer 升 Professional | 选模板 + 自然语言改 + 简单拖拽;为自家定制 Persona | Canva 模板化 |
| **L3 完整创作** | Professional / ACB | 全 Forge 能力 + 画板 2D/3D + Persona 库 + 驱动开发 | Figma / UniFi Design |
| **L4 商业交付** | ACB | + 客户档案 / 合同 / 项目管理 / 持续服务 / Lead 派单 + 分润 | Houzz Pro |

**关键**:L1→L2 是"我想自己改改"门槛极低;L3→L4 是"我接单了需要工具管理客户"门槛也低。每一跳都是自然需求驱动,不是强制升级。

## 三、Forge 容器架构

### 3.1 Workshop = 一组可插拔 Forge

```
Builder Workshop(embedded capability set)
   │
   ├── Space Design          ★ MVP 主力 — 售前空间方案(户型/点位/设备/自动化/可视化)
   ├── Life Dashboard Builder ☆ MVP 并行 — 已有 Studio/Home 数据后的 Aqara Life 看板生成
   ├── Protocol Driver Agent  ☆ MVP 并行 — MQTT/HTTP/TCP/UDP 协议文档解析 + 驱动生成 + Studio 部署
   ├── Plugin Builder         ☆ MVP 并行 — 看板组件、服务插件、通用插件开发,可自用或发布
   ├── Driver Forge           ☆ 后续扩展 — Modbus/BACnet/KNX 等复杂协议接入
   ├── Scene Forge   — V2                — 场景与自动化
   ├── Voice Forge   — V2                — 语音脚本
   ├── Care Forge    — V2                — 健康/育儿/适老专项
   ├── Energy Forge  — V2                — 能源管理
   └── ...           — V3 第三方可上架    — Forge SDK 开放后
```

### 3.2 内核 = 5 件公共能力

每个 Forge 共享同一套底层服务,新增 Forge 只需注册 Manifest + 实现编辑器 UI:

```
┌──────────────────────────────────────────────────┐
│ Workshop Kernel (公共能力,所有 Forge 复用)        │
├──────────────────────────────────────────────────┤
│ 1. Context Container — Project / Home / Plugin  │
│ 2. Copilot Bus       — AI 能力 (Spatial IQ +     │
│                        Vision + LLM Gateway +    │
│                        Device Graph + Persona +  │
│                        Protocol IQ)              │
│ 3. Virtual Studio    — 沙箱预览(时间穿梭/        │
│                        多视角/虚拟设备/断网模拟) │
│ 4. Publish Pipeline  — 五档发布 + 审核流          │
│ 5. Forge SDK         — Manifest + 注入点 +        │
│                        权限 + i18n + 分成         │
└──────────────────────────────────────────────────┘
```

### 3.3 Forge SDK Manifest(关键字段)

```yaml
forge.manifest.yaml
├── meta: id, name, version, icon, author, category, maturity
├── i18n: locales, translations (name/desc/tutorial)
├── inputs: types, max_size, allowed_mime
├── outputs: types, target_runtime
├── copilot:
│   ├── required: [spatial_iq, device_graph, ...]
│   ├── optional: [vision]
│   └── own_model: false
├── sandbox:
│   ├── permissions
│   ├── quota: cpu_ms / memory_mb / timeout_s
│   └── data_residency: [cn, eu, me, kr, ...]   # 合规硬约束
├── preview:
│   ├── engine: virtual_studio | web_viewer | device_emulator
│   └── entry: /preview.html
├── lifecycle: onInstall / onApply / onFeedback / onUninstall
├── monetization: model (free|paid|rev_share), price, split_ratio
└── compliance: review_status, reviewer, regions_available
```

**关键设计点**:
- `data_residency` 是强约束(平台强制路由,不是声明)
- `copilot.required` 是治理工具(平台审查为何要用 device_graph)
- `maturity` 三档(experimental / verified / commercial)对应五档发布

### 3.4 第三方 Forge 开放节奏

| 阶段 | 谁可开发 | 审核 | 类比 |
|---|---|---|---|
| MVP 0-6m | 仅 Aqara 官方 | 内部 | Apple 早期自营 |
| 6-12m | Top 50 ACB(邀请制) | 人工 + 自动扫描 | YouTube YPP 早期 |
| 12-24m | 所有 Verified ACB | 自动 + 抽检 | Figma Plugins |
| 24m+ | 外部公司 / 开发者 | 严格审核 + 分成 | Apple App Store |

> ⚠ MVP 阶段绝不开放第三方 Forge — Studio 是本地 OS,出一次安全事故就是品牌灾难。

## 五、五段统一生命周期

```
[1 发起]   多模态: 自然语言 / 上传户型 / 模板库 / Fork 别人项目
              ↓
[2 工作台] 三层 AI 交互(借 Cursor):
            • Inline   — 光标处微操作("选中房间 → 推荐设备组合")
            • Chat     — 侧栏对话("这个户型适合老人居住吗")
            • Composer — 多对象批改("重做整个两居室 Persona")
              ↓
[3 沙箱]   Virtual Studio(Aqara 独有护城河):
            • 时间穿梭(预览早上 6 点场景)
            • 多身份视角(家长 / 老人 / 访客同屏对比)
            • 断网模拟(本地优先降级体验)
            • 虚拟设备池(所有 Aqara SKU 的 digital twin)
              ↓
[4 发布]   五档: Draft / Private / Published / Verified / Commercial
              ↓
[5 Apply]  简单方案 → 自助部署 Studio(若有授权)
            复杂方案 → 派单算法(同城作者优先,否则 3 候选)
            跨城 → 远端作者 review + 本地 ACB 施工协作订单(V2)
```

### 5.1 五档发布

| 档 | 可见 | 审核 | 商业化 | 类比 |
|---|---|---|---|---|
| Draft | 仅本人 | — | — | 草稿 |
| Private | 链接分享 | — | — | Notion Private |
| Published | Community 社区(MVP 不开,V2 启用) | 自动扫描 | 打赏 + Boost | Steam Workshop |
| Verified | + Aqara 认证标识 + 推荐位 | 人工审核 | Lead 派单优先 + 原作者分成 | Apple Editor's Choice |
| Commercial | 全球推广位 + Marketplace | 深度评审 + 商务谈 | 付费应用 + 订阅 + 深度分成 | Lego Ideas 上架 |

> MVP 只开 Draft / Private / Verified 三档。

### 5.2 Apply 部署的两条路径

```
作品 / 项目页 [Apply 按钮]
       │
       ▼
[分流判断]
   ├─ 简单方案(<10 设备 + 标准自动化 + 无空间改造)→ 自助 Apply 到 Studio
   └─ 复杂方案(全屋布线 + Persona 定制 + 跨房联动 + 需上门)→ 触发 Lead
                                                                ↓
                                                          派单算法(见第十节)
```

**简单 vs 复杂的判定**(MVP 平台 + 作者共同标注,V2 用规则引擎):
- 设备数量与品类(传感器 vs 面板 vs 灯控布线)
- 是否需要空间测量 / 布线改造
- 作品的 Persona 个性化依赖度
- 作者标记 "Require Professional Installation"

## 六、Space Forge — MVP 首发主力

### 6.1 端到端流程(8 步 AI 交互)

```
Step 1  发起(任选其一)
        ├─ 上传户型图(JPG / PNG / PDF)
        ├─ 上传 CAD 文件(.dwg / .dxf)
        ├─ 自然语言描述("140m² 三室两卫,老人住主卧")
        └─ Fork 别人作品(灵感库)
              ↓
Step 2  AI 户型识别(Spatial IQ + Vision)

        └─ 用户确认或修改
              ↓
Step 3  基础设定 + 风格主题
        ├─ 家庭成员(数 + 年龄 + 角色)
        ├─ 预算 + 智能化深度
        ├─ 风格主题(关爱老人 / 科技极客 / 极简 / 亲子 / 奢华度假)
        └─ 文化偏好(国家 + 语言)
              ↓
Step 4  AI 一键生成"分镜"(对应 6 个 Persona 场景预设)
        ├─ 早晨起床
        ├─ 老人用药
        ├─ 小孩放学
        ├─ 家长下班
        ├─ 夜间安防
        └─ 周末聚会
              ↓
Step 5  Persona 配置(每个成员的千人千面)
        ├─ AI 推荐(基于年龄 + 角色 + 风格)
        └─ 用户拖拽 / 自然语言微调
              ↓
Step 6  设备布局(画板 2D/3D)
        ├─ AI 自动推荐设备组合 + 位置 + 数量
        ├─ 用户拖拽调整
        └─ 实时显示覆盖范围 / 信号强度 / 物理冲突
              ↓
Step 7  Virtual Studio 沙箱预览
        ├─ 时间穿梭(老父早起 6 点 / 全家欢聚 18 点 / 深夜警戒 2 点)
        ├─ 多身份视角(家长 / 老人 / 访客 同屏对比)
        └─ 断网模拟(本地优先降级体验)
              ↓
Step 8  发布或 Apply
        ├─ 发布到 Verified 档 → 等待 Aqara 审核
        ├─ 自助 Apply 到自家 Studio(若简单方案)

```

### 6.2 输出物 — 空间图谱(Spatial Graph)

Step 8 部署到 Studio 时,输出一份 **空间图谱**,包含:
- 房间拓扑 + 尺寸 + 门窗
- 设备清单 + 位置 + 协议绑定
- 自动化规则 + 触发条件
- Persona 配置 + 场景模式
- 时间触发 + 地理围栏 + 物理传感

Studio 加载图谱后即可运行,客户家不需要重新配置。

## 七、App Forge / Driver Forge(MVP 并行)

### 7.1 App Forge — App 页面 vibe 创作

- 输入:自然语言描述 / 上传图片(参考样式)/ 选择设备类型 / 选择服务类型
- 输出:Aqara Life 内的家庭定制页面
- 沙箱:Web Viewer + Mock 家庭上下文(时间 / 在家人数 / 设备状态)
- 关键场景:为老人定制"大字体语音首页",为儿童定制"家长管控首页",为客户定制品牌页面

### 7.2 Driver Forge — 协议接入 + 驱动生成

- 输入:协议类型(MQTT / Modbus / Zigbee / Matter / 私有协议)+ 上传协议文件(PDF / 文档)
- 输出:JS 驱动库 + 设备模板 + 测试用例
- 沙箱:Device Emulator(伪造设备响应,跑完整"配对 / 控制 / 异常"用例)
- 部署:一键到 Studio,客户端只需"发现设备"

> Driver Forge 也是 Aqara **开放生态战略入口** — 第三方设备厂商通过 Driver Forge 快速接入,降低 Aqara 自营驱动覆盖压力。

## 七B、Plugin Builder — 插件开发者画布

> **Plugin Builder 是面向开发者的专属创作工作台**，从 Build AI 对话框一键进入，支持四类插件类型画布切换，AI 辅助生成代码 + 可视化调试 + 一站式发布/分享。
> **与 Driver Forge / App Forge 的区别**：Forge 产出的是项目级私有资产(只用于特定方案)，Plugin Builder 产出可打包、可发布、可多用户复用的插件包。

### 7.3 四类插件类型

| 插件类型 | 典型场景 | 核心输入 | 产出物 |
|---|---|---|---|
| **设备协议插件** | 接入第三方非 Aqara 设备 | 协议类型(MQTT/HTTP/UDP/TCP) + 设备品类 + 描述 | JS 驱动包 + 设备模板 + 测试用例 |
| **服务类插件** | 安防联动 / 节能策略 / 健康监护 | 服务类型 + 触发条件 + 响应动作 | 自动化规则包 + API 集成脚本 |
| **App 设备详情页插件** | 自定义设备卡片/详情页 UI | 目标设备类型 + UI 风格 + 功能模块 | React 组件包 + Mock 数据 |
| **App 看板插件** | 多设备统览 / 能耗 / 安防大屏 | 看板类型 + 数据源 + 布局描述 | Dashboard 组件包 + 数据绑定配置 |

### 8.2 Aqara 独有护城河 — Studio 物理锚点

```
       ┌───────────────────────────────────────────────┐
       │    ACB 创作精品(Verified / Commercial)       │
       │    ← 风格主题 + Persona 模板 + AI Copilot     │
       └───────────────────────────────────────────────┘
                          ↓
       Aqara Life "灵感"频道(抖音式浏览 + 3D 预览)
                          ↓
       AI 推荐:"和你家匹配 82%"
                          ↓
       ┌──────────────── 分流 ────────────────┐
       │                                       │
  简单方案 ✓                              复杂方案 ✓
  自助 Apply                              触发 Lead 派单
  → 部署 Studio                           → ACB 接单 + 交付 + 分润
                          ↓
       使用数据回流(匿名脱敏聚合)
                          ↓
       ├── AI 推荐更准(Spatial IQ 飞轮)
       ├── 原作者仪表板反馈(使用率 / 关闭率 / 触发率)
       ├── 作者迭代 → 新版本推送 → 客户可选升级
       └── 全球作品热度榜(元数据跨境 + 运行时本地)
```

**Aqara 比 Figma / Midjourney 强的差异化**:

- 跨数据中心联邦飞轮(原始数据不跨境、元数据 + 模型参数可跨境,学 Google Federated Learning)

## 九、创作者经济 — 只给非现金激励

战略选择:**MVP 不投现金创作者基金,只给非现金激励**。理由:
- 现金基金有刷量 / 预算天花板 / 停发流失等隐形成本
- 非现金激励把"激励 ↔ 业务价值"死死绑在一起
- 学 Apple Developer / AWS Hero / GitHub Stars 路径

### 9.1 9 类激励的"价值阶梯"

| 激励 | 对 ACB 价值 | 杠杆 | 机制 |
|---|---|---|---|
| ★ **Lead 派单优先级加权** | 直接影响现金收入 | 最高 | Verified 作品被 Apply 次数 × 作者接单权重加成 |
| ★ **Aqara 官方 Lead 指派权** | 大客户直派(展会 / 地产 / 企业) | 极高 | 顶级 ACB 专属,每年 ≤10 名额 |
| ★ **等级返利加速** | 业绩返利阶梯提升 | 高 | 作品贡献计入 ACB 等级评定维度 |
| Aqara Academy 学分 | 抵学费 / 升级 Badge / 解锁高阶 | 中高 | 直接对接现有 Academy 体系 |
| Aqara 设备折扣 | 自用 + 赠客户 + 做样板间 | 中 | 分段折扣,头部 ACB 可拿批发价 |
| Aqara Life / Builder 推广位 | 流量曝光 | 中 | 灯塔三国独立榜 + 区域精选位 |
| Aqara 官方媒体宣传 | 品牌背书 | 中 | 案例采访 / 官方社媒 Feature / 发布会样板 |
| 海外发展优先权 | 全球化新市场进入 | 低中(长期高) | 灯塔后新国家开时,头部 ACB 优先成为门店主候选 |
| 早期 Forge SDK 测试资格 | 未来第三方 Forge 开放时头部位 | 长期高 | Top 50 ACB 邀请制 |

每一项最终都能折算成现金(更多 Lead → 更多服务费;设备折扣 → 利润空间;Academy 学分 → 升级更高分润档),但路径是**业务路径**,Aqara 不花一分钱"补贴"。

### 9.2 冷启动三阶段策略

| 阶段 | 时间 | 目标 | 策略 |
|---|---|---|---|
| **种子** | 0-3m | 50-100 个标杆作品 | Aqara 核心团队全员产出,覆盖典型户型 × 典型 Persona;占据灯塔三国首页 |
| **灯塔 ACB 荣誉计划** | 3-9m | 30 名 Founder Creator | 每灯塔国(韩 / 迪 / 米)各 10 名;给"门店主优先权 + Aqara 官方 Case Study + Academy 进阶免费 + 跨国大会邀请";要求 6 个月产出 ≥5 Verified 作品 |
| **内容飞轮自转** | 9-18m | 内生增长 | 头部 ACB 吃 Lead 分润 → 自发产出;长尾靠 AI 匹配推荐 + 编辑精选兜底 |

> **不要跳过种子阶段**:空平台永远起不来,Aqara 团队亲自下场三个月是必须的投入。

## 十、作品 → Lead 派单链路

> 详细分润 / 派单 SOP 见 [`../03-architecture/service-billing-model.md`](../03-architecture/service-billing-model.md) §五分润 / §九挂靠 / §十海外。本节聚焦 **作品作为 Lead 入口** 的产品设计。

### 10.1 三层漏斗:作品页 → Lead 转化

```
作品页浏览
  ↓
[Apply 按钮] 或 [我也想要]
  ↓
分流判断(简单 vs 复杂)
  ├─ 简单 → 自助 Apply
  └─ 复杂 → 触发 Lead
            ↓
       派单算法(MVP "3+1 候选 + 客户选")
            ↓
       客户选 ACB → 报价 → 签约 → 部署 + 分账
```

### 10.2 派单算法关键维度(MVP)

**Tier 1 硬筛**(必须全命中):
- 服务半径(ACB 常驻城市 + 50km;海外按国家代理辖区)
- ACB 等级 ≥ 作品难度等级(L1 作品任何 ACB 可接,L3 作品必须资深)
- ACB 当前接单容量未满

**Tier 2 软排序加权**:
- **原作者同城度**(本人同城 → 权重最高,直接置顶)
- 作品风格 / Persona 领域匹配度(适老 / 极客 / 极简 ...)
- 历史评分与完工率
- 客户预算与 ACB 报价区间匹配
- 挂靠 Aqara Space 门店的地理协同

候选池展示:**3 个候选 + 原作者(若同城)**,客户自选或"让平台帮我选"。

### 10.3 原作者分润(防"抄作业")

| 角色 | MVP(0-12m) | 长期(12-36m) |
|---|---|---|
| 本地履约 ACB | 70% | 70-75% |
| **远端原作者(IP 费)** | **8%** | **5-10%(按热度浮动)** |
| 挂靠 Aqara Space 门店 | 10% | 8-10% |
| 平台抽成 | 5% | 5-7% |
| Aqara 品牌保留金(售后兜底) | 7% | 3-5% |

**关键设计**:
- 原作者分润**只在"作品被完整采用"时触发**(本地 ACB 重度改造 >40% 时降为 2-3% 或 0)
- 原作者若**同城且本人接单**,直接拿 70%,不叠加 IP 分成(防双重获利)
- 作者创作时选三档身份:"只创作不接单" / "同城接单" / "全国可接单(远程指导)"

### 10.4 跨城远端作者 + 本地 ACB 协作订单(V2)

借短剧 IP 跨地拍摄 + 医疗"异地专家 + 本地医生"会诊模式:
- 客户在北京看到上海 ACB 作品
- 平台生成"协作订单":上海 ACB 远程方案 review(15-20% IP 费) + 北京 ACB 实地施工(55-60%)
- 客户感受到"原作者亲自远程把关"的增值信任

> 这是 Aqara **结构性护城河** —— 普通装修 / 家电平台都做不到。MVP 不上,V2 启用。

### 10.5 脱平台治理 — 工具绑架 > 规则惩罚

Studio 是物理锚点 = 天然反脱平台:
- 作品部署走 Studio → Builder Console → 平台系统,脱平台无法部署
- 售后联动(设备报警 / 远程调校)必须走平台
- ACB 等级升级 / 积分 / 客户好评全部绑平台账号

**MVP 阶段**:平台不主动查脱平台,但所有 Studio 部署留痕,脱平台作品**无售后、无保修、无积分**。
**长期**:客户在平台完整交易可获"Aqara 官方质保延长 2 年"等硬权益,增加脱平台成本。

## 十二、全球化扩展

### 12.1 元数据跨境,运行时本地

合规约束:中国 / 欧洲(GDPR)/ 韩国(K-PIPA)/ UAE 各自数据驻留硬约束。Aqara 飞轮采用 **联邦式而非中心式**:

```
        [区域 DC: CN/EU/KR/ME]                  [全球 Bus]
┌────────────────────────────────┐    ┌──────────────────────┐
│  ACB 创作 Forge 作品           │    │                      │
│       ↓                        │    │                      │
│  其他 ACB Apply 到客户 Studio  │ ─→ │ 元数据 Only         │
│       ↓                        │    │ (不含客户隐私)       │
│  客户真实使用行为              │    │                      │
│       ↓                        │    │ ↓                    │
│  本地模型(Persona/推荐)微调  │ ←─ │ 全球 Forge 热度榜  │
│       ↓                        │    │ 跨区作品发现(脱敏) │
│  反哺本区 Bus / Device Graph   │    │ Device Graph 共享更新 │
└────────────────────────────────┘    └──────────────────────┘
```

**三类数据分级**:
- 原始客户数据(绝不跨境)
- 作品元数据(可跨境)
- 模型参数增量(联邦学习跨境)

### 12.2 ACB 全球护照

- ACB 跨国移居时,作品集展示层跟随 Badge 跨境
- 但客户评价 / 星级需要分区显示(韩国 4.8 ≠ 米兰 4.8)
- 类比 LinkedIn Global Profile + 国际驾照分区认可

### 12.3 区域独立热度榜

不强推"全球 Top 10",学 Netflix 各国本地化 — **韩国 / 迪拜 / 米兰各有独立精选榜**。文化差异(米兰偏极简、迪拜偏奢华、韩国偏科技极简)通过风格主题显性化。

### 12.4 跨境作品 Apply 三步授权

```
韩国 ACB 作品 → 米兰 ACB 想 Apply 给客户
   ↓
1. 韩国作者授权(可设默认许可,免重复授权)
2. 米兰 ACB 做本地化适配(设备 SKU 重映射 / 语言 / 合规)
3. 客户合规绑定(GDPR 同意流)
```

## 十四、落地原则

1. **项目是一等公民,不区分自用 / 客户** — 部署授权和可见性是正交维度
2. **同一套 UI,身份渐进解锁** — 不做 4 套 UI,L1→L4 自然过渡
3. **AI 算力档跟身份正交** — 用户按任务自由选(总设计师 / 方案师 / 草图师)
4. **AI 漫剧"导演工坊"心智** — 不要用"设计器 / IDE"等工具词,用"项目 / 角色 / 分镜 / 风格"叙事
5. **Workshop Kernel 5 件公共能力,Forge 共享** — 新增 Forge 走标准 SDK,不重复造轮子
6. **MVP 绝不开放第三方 Forge** — Studio 出一次安全事故就是品牌灾难,12m+ 再开
7. **Studio 物理锚点是反脱平台的根本** — 作品必须部署 Studio 才生效,工具绑架 > 规则惩罚
8. **MVP 不投现金创作者基金** — 9 类非现金激励 + 业务路径变现
9. **持续服务是 Aqara 独有护城河** — 项目永不结束,V1 基本 V2 全面
10. **联邦式数据飞轮,不是中心式** — 元数据跨境 + 模型参数跨境 + 原始数据本地驻留
11. **跨境派单 MVP 禁用** — V2 远端作者 + 本地 ACB 协作订单是结构性护城河
12. **AI 漫剧式 UX 一定要有** — 一键生成分镜 / 风格主题 / Copilot 三层交互 / 时间穿梭沙箱
13. **Plugin Builder 是独立画布,不是对话框附属** — 插件开发有独立 Canvas 工作区,四类插件各有专属配置+调试视图
14. **插件生命周期完整**:开发 → 自用/分享/上架 三条路径并重,二维码分享是低门槛扩散关键

## 十五、Plugin Marketplace — 插件生命周期与应用路径

> 插件开发完成后，面对两类主体：**开发者**（决定怎么用）和**用户**（从哪里获取）。

### 15.1 开发者路径 — 三选一

```
Plugin Builder 开发完成
       │
       ├──► 自用 — 直接部署
       │         ├─ 部署到自己的 Studio（协议插件 / 服务插件立即生效）
       │         └─ 推送到 Aqara Life App（App 详情页 / 看板插件）
       │
       ├──► 私密分享（不发布）
       │         ├─ 生成二维码 → 对方扫码安装到 Studio 或 Aqara Life
       │         └─ 生成私有链接 → 发给特定用户/团队
       │         ⚠ 此路径不经审核，作者自负其责，无平台背书
       │
       └──► 发布到 Marketplace
                 ├─ 定价：免费 / 一次买断 / 月订阅 / 年订阅
                 ├─ 审核：MVP 人工审核；V2 自动扫描 + 抽检
                 ├─ 分成：付费插件 开发者 70% / 平台 30%
                 └─ 上架后可随时更新版本（用户可选升级）
```

### 15.2 用户路径 — 获取与应用

```
Marketplace 浏览 / 搜索 / AI 推荐
       │
       ▼
[获取插件]（免费/付费）
       │
       ├──► 应用到 Aqara Studio（部署到家庭本地节点，协议插件/服务插件即时生效）
       │
       ├──► 扫码安装到 Aqara Life App（C 端）
       │         → 看板插件 / 设备详情页直接出现在 App 内
       │
       └──► 收藏备用（已购不重复付费，随时启用）
```

### 15.3 Marketplace 设计要点

| 维度 | 设计 |
|---|---|
| 插件分类 | 设备协议 / 服务 / App 详情页 / App 看板 |
| 插件卡片 | 图标 + 名称 + 作者 + 评分 + 安装数 + 价格 |
| 兼容性标注 | 支持的协议版本 / 最低 Studio 固件 / 适配 Aqara Life 版本 |
| 安全等级 | 🟢 官方认证 / 🔵 Verified ACB / ⚪ 社区发布 |
| 二维码安装 | 扫码直达插件详情页 → 一键安装(需登录 Aqara 账号) |
| 沙箱隔离 | 插件运行在受限沙箱内，不可访问未授权 Studio 数据 |
| 退订退款 | 年订阅未超 30 天可退款；买断型不退(前有试用期) |

### 15.4 MVP / 长期路线

| 阶段 | 开放策略 |
|---|---|
| **MVP** | Plugin Builder 仅对 ACB / Pro 开放；Marketplace 仅展示官方 + 受邀开发者插件 |
| **6-12m** | Top 50 ACB 邀请制发布插件；二维码私密分享全量开放 |
