# Service Billing & Operations Model

> ACB 接单 → 平台托管 → 三方履约 → 钱包结算 的端到端机制。
> 三条硬约束:用户永远直付 Aqara(防 ACB 跑路)、货服分离(官方仓直发用户家)、Aqara Space 门店准入由总部独家审批(代理不得养亲兵)。
> **挂靠规则全球统一**:ACB 身份归 Aqara 总部、关系归 Aqara Space 门店,Builder Console 一套代码跑国内外。海外多一层国家代理 / Operator-Store 作为运营治理叠加,代理可换不影响门店和 ACB。米兰首店因意大利暂无国家代理,采用 **Operator-Store(门店即运营层)** 过渡模式。
>
> **当前实现状态**:本文档为目标态架构。MVP 范围在最后一节明示。

## 二、ACB 接单类型与计费形态


| 形态         | 场景                      | 计费模式       | 释放节奏                       | MVP 是否上线 |
| ---------- | ----------------------- | ---------- | -------------------------- | -------- |
| **安装一次性单** | 新房装解 / 设备安装 / 迁居重装      | 一次付清入托管    | 货款签收即放 + 工时验收即放 + 质保金 30 天 | ✅        |
| **巡检包月**   | 大宅 N 次上门 / 季度巡检         | 预付服务券,按次核销 | 单次完工核销释放                   | ✅        |
| **长期托管订阅** | Persona 长期优化 + 远程托管     | 月费扣款入托管    | 月度滚动释放 + 投诉窗口              | V2       |
| **紧急维护**   | Temporary Service 时间窗诊断 | 单次入托管      | 完工即放                       | V2       |


### 关键设计

- **设备款与服务费分离结算**:即使用户一次付清,后台拆 3 段(设备款 / 服务费 / 平台技术费),分别归属,分别释放。
- **本地优先承诺不变**:服务交付完成后,Studio 本地运行不依赖订阅持续 — 用户停止订阅 ACB 服务后,Persona 配置仍永久保留在本地 Studio。云端是增值代理,不取代本地。

## 三、用户付款 & 平台托管(Escrow)

### 3.1 硬约束:用户永远直付 Aqara,不直付 ACB

参考 滴滴(司机不收现金)/ 美团到家(平台收单)/ 瑞幸(App 内闭环)。这条是防 ACB 卷款跑路的根本机制,所有形态都遵守。

### 3.2 托管账户结构

- **Customer Escrow Wallet**:用户付款后进入平台托管账户,**按订单/项目独立子账户**,资金专款专用。
- **资金性质**:在用户验收前是预收账款(平台负债),不是 Aqara 收入,不是 ACB 收入。
- **隔离规则**:跨数据中心的资金不互通;海外部分由当地 MoR 主体独立托管。

### 3.3 三段式释放规则


| 触发器         | 释放比例                | 时间     | 类比             |
| ----------- | ------------------- | ------ | -------------- |
| 用户主动确认验收    | 立即放 80%(货款 + 工时款)   | T+0    | 淘宝"确认收货"       |
| 自动验收(超时未投诉) | 自动放 80%             | T+7 天  | 淘宝默认释放         |
| 评分 ≥ 4 星    | 触发 +5% 质量奖金(平台保险池出) | T+0    | 滴滴服务分激励        |
| 30 天质保期满    | 释放剩余质保金             | T+30 天 | Houzz Pro 项目尾款 |


### 3.4 验收期与质保金水位(本批次定稿)


| 订单类型             | 验收期 | 质保金水位    | 质保期  |
| ---------------- | --- | -------- | ---- |
| 标准单              | 7 天 | 10%      | 30 天 |
| 大额单(> ¥20,000)   | 7 天 | 20%      | 60 天 |
| 高等级 ACB(L4/L5)接单 | 7 天 | 5%(等级激励) | 30 天 |


### 3.5 异常处理

- **用户投诉**:冻结待释放余额,转人工仲裁队列
- **用户退款**:走原路退回,平台在 ACB 钱包对应金额做反向冲账;若 ACB 钱包余额不足,从保证金扣;再不足走保险池兜底
- **ACB 跑路**:**平台 7 天内代赔用户全额**,ACB 列入黑名单,联合追偿(详见第十二节)

## 四、ACB 钱包 & 提现规则

### 4.1 4 档余额模型


| 余额        | 含义                   | 可提现           |
| --------- | -------------------- | ------------- |
| **冻结中**   | 订单未完工 / 验收期内 / 投诉处理中 | ❌             |
| **待结算**   | 已验收但在保留金期内           | ❌             |
| **可提现**   | 全部释放完毕               | ✅             |
| **保证金账户** | 业绩换押金累积,独立子账户        | ❌(违规扣 / 退出时退) |


### 4.2 提现规则


| 维度   | MVP 阶段                    | 长期目标态                   |
| ---- | ------------------------- | ----------------------- |
| 到账周期 | T+1 工作日                   | D+0 实时(L4+ 福利)          |
| 单日限额 | ¥50,000                   | 等级阶梯(L1=¥20k, L5=¥200k) |
| 手续费  | 千分之三(对标支付宝商家)             | L4+ 免手续费                |
| 风控触发 | 大额 / 夜间 / 新设备 / 异地 → 人工审核 | 同 + AI 异常画像             |


### 4.3 海外提现

- 走 **Stripe Connect / Wise for Platforms**,ACB 在 Builder Console 绑定本地银行 / PayPal / Wise 账户
- **国家代理不得过手 ACB 资金**(红线)— 哪怕过渡期国家代理作为 MoR 收款,ACB 分润也走平台直付
- 当地货币结算,汇率与手续费透明展示

## 五、ACB 等级体系 & 分润 & 返利

### 5.1 架构层定义(具体级数交给运营规则)

- 架构层支持 **N 级动态配置**(运营可调整级数 / 比例 / 阈值,无需改代码)
- 等级影响 4 件事:**派单优先级 / 分润比例 / 设备采购折扣 / 返利**
- **国内服务商接入时**:历史业绩为冷启动等级依据(详见第十三节迁移)

### 5.2 等级评定维度


| 维度                   | 权重示意 |
| -------------------- | ---- |
| 完工评分(用户给)            | 高    |
| 投诉率 / 服务分            | 高    |
| 业务量(月度)              | 中    |
| 培训完成度(Academy Badge) | 中    |
| 推荐插件销量(被动)           | 低    |
| 服务覆盖区域               | 低    |


- 评定周期:月度刷新,季度生效
- 升 / 降级:都自动,人工只受理申诉
- 保级:连续 N 月未达标自动降级

### 5.3 单笔订单分润切分(基础结构)

```
用户付款(以服务费部分为例,设备货款另算)
  │
  ├─ ACB 主力分润                70% – 85%(等级阶梯)
  ├─ 平台抽成                     5% – 10%
  ├─ 挂靠 Aqara Space 门店分成     5% – 10%(国内)
  ├─ 国家代理通道分润              0(国内)/ ≤ 2pp(海外,封顶,从平台抽成切)
  ├─ 保险池注资                    0.5% – 1%(从平台抽成切)
  └─ 等级返利池预留                季度返点资金来源
```

### 5.4 三层返利节奏


| 节奏       | 形态                             | 触发        |
| -------- | ------------------------------ | --------- |
| **每单实时** | 跟随订单释放,即时入 ACB 钱包              | 验收 / 自动释放 |
| **季度返点** | 累计销售额阶梯返,Q+1 月 15 日发放          | 季末业绩盘点    |
| **年度奖**  | Top 10% ACB 额外奖金 + 实物激励 + 大会邀请 | 年度盘点      |


### 5.5 海外国家代理三层分润(对标 Apple AASP / Loxone Country Partner)


| 分润类型                             | 比例        | 来源   | 触发                               |
| -------------------------------- | --------- | ---- | -------------------------------- |
| **货款通道分润**(Transaction Override) | 3% – 5%   | 设备货款 | 本国 ACB 在 Builder Console 下 PO 出货 |
| **项目激活分润**(Project Bonus)        | 1% – 2%   | 项目金额 | Project Pricing 大单完工验收           |
| **服务抽成的小头**(Service Override)    | ≤ 2pp(封顶) | 平台抽成 | 本国 ACB 服务订单结算                    |


> 与 ACB 等级返利**完全平行账本**,代理拿多少与 ACB 拿多少互不感知互不冲突。

## 六、保证金 + 保险池 + 服务分(治理三件套)

### 6.1 保证金:业绩换押金模式(本批次定稿)

- **不预收押金**:前 N 单冻结尾款的 5%,累积到等级门槛
- 门槛随等级递增(L1 ¥3,000 / L3 ¥5,000 / L5 ¥10,000)
- L4+ 信用良好可申请免缴
- 退出 ACB 体系 + 无未结纠纷 → 全额退还

> 学滴滴 2020 后"业绩换押金"模式,降低准入门槛 — 但风控前置不足,需保险池兜底。

### 6.2 保险池:共担风险池

**资金来源**:

- 平台抽成的 0.5%-1% 持续注入(MVP: 0.5%)
- ACB 每单服务费的 0.3% 强制扣缴(MVP 暂不上,V2 启用,届时包装为"职业保险")
- 平台年度补充注资(兜底)

**赔付场景**:

- ACB 跑路 → 保险池先垫付用户,再向 ACB + 保证金追偿
- 安装质量事故(漏水 / 触电)→ 保险池 + ACB 保证金共担
- 设备假货赔付 → 全平台兜底
- 海外可包装为产品责任险(Lloyd's / Chubb 真保险公司承保),平台只做投保人

### 6.3 服务分(Service Score):0-100 动态信用分

- 起始分 80
- 每单评分:5 星 +1 / 4 星 0 / 3 星 -2 / 2 星及以下 -5
- 投诉成立 -10,严重投诉(假货 / 卷款 / 事故)-30
- 30 天滚动累计,影响下季度等级

### 6.4 三档处罚(违规扣款)


| 档位        | 触发条件              | 处罚                                      |
| --------- | ----------------- | --------------------------------------- |
| 🟡 **黄牌** | 单笔差评 / 服务分跌至 60   | 警告 + 降派单优先级 + 罚款 ¥100 – 500             |
| 🟠 **橙牌** | 单笔严重投诉 / 服务分跌至 40 | 暂停接单 7-30 天 + 强制重培训 + 罚款 ¥1,000 – 5,000 |
| 🔴 **红牌** | 跑路 / 假货 / 安全事故    | 永久封号 + 保证金全没 + 进入行业黑名单 + 法律追究           |


**关键设计**:

- 赔付优先级:用户损失先扣 ACB 钱包余额 → 不够扣保证金 → 再不够走保险池
- 自动降级 + 人工申诉前置(海外 GDPR / 加州 AB5 等劳动者保护法硬要求)
- 头 6 个月宽松起步,第二年开始收紧 — 学滴滴 / 美团治理路径

## 七、PO 流程 & 货物流转

### 7.1 主路径:货服分离(占 85% 订单)

> **官方仓 / 国家代理仓 / Aqara Space 门店仓 → 直发用户家;ACB 空手上门安装。**

参考 京东送装一体 / 装修平台主材包 / 美团闪购 / IKEA Assembly。**ACB 不持货 = 假货风险归零 + 卷款风险归零**。

### 7.2 例外路径:白名单备件(V2 才开,MVP 不开)

- 仅 L3+ Installer 可备
- 清单:50 SKU,单件 ≤ ¥500,总值 ≤ ¥10,000
- 序列号必须从 Aqara 官方渠道采购,系统强校验
- 季度盘库,差异 > 2% 降级处理

### 7.3 PO 三段扣款

```
T0  接单 / PO 创建
    │
    ├─ Aqara 仓硬预占 SKU(Hard Allocation,72h 上限)
    └─ 用户托管账户冻结 70%(货款) + 20%(工时) + 10%(质保金)

T1  仓库出库
    │
    ├─ 货款部分(70%)从"冻结"转"待结算"
    └─ ACB 端不扣款(因为 ACB 不持货)

T2  签收 + 验收
    │
    ├─ 用户签收 → 货款结算给 Aqara(内部记账)
    ├─ 服务验收 → 工时款释放给 ACB 钱包
    └─ T+30 质保期满 → 质保金释放给 ACB 钱包
```

### 7.4 跨数据中心订单(关键约束)

- **Master Order ID 由下单域签发**,一订单一域,不跨境同步单据
- 跨境项目(极少)拆单成两个同业务域的关联订单,用 `correlation_id` 串联
- ACB 身份跨域认证靠 Aqara Academy 证书的 JWT 互信
- 详见 `[./data-model.md](./data-model.md)`

### 7.5 海外项目大单通道(Project Pricing)

参考 Loxone Project Pricing / Apple AASP Parts Portal:

- **触发**:单笔项目超过阈值(€10,000 或 50+ 设备),ACB 发起 Project Registration
- **审批**:Aqara 总部 + 国家代理联合审核(SLA 48 小时)
- **履约**:海外仓直发(标品仍走代理仓)
- **价格**:Aqara 总部统一设定 Project Pricing,代理不能调

## 九、挂靠关系:ACB ↔ Aqara Space 门店(全球统一规则)

> ACB 身份归 Aqara 总部、关系归 Aqara Space 门店。
> 全球采用同一套挂靠规则,Builder Console 一套代码跑国内外。
> 国家代理 / Operator-Store(海外才有)是治理叠加层,不改变挂靠关系本身。

### 9.1 统一挂靠规则


| 维度           | 规则                                                                       |
| ------------ | ------------------------------------------------------------------------ |
| 身份所有权        | ACB 身份归 Aqara 总部(全球唯一)— 与挂靠门店无关                                          |
| 关系性质         | 多对多(一个门店挂多个 ACB,一个 ACB 可挂多门店或换门店)                                        |
| 挂靠分成         | Aqara Space 门店从 ACB 服务费分润里切 **5–10%**                                    |
| 触发条件         | 订单地理位置在门店辐射范围 / 客户来自门店导流 / ACB 主动选择挂靠                                    |
| 解绑机制         | **90 天冷静期** — 解绑后 90 天内的存量订单仍按原分成,新订单按新挂靠                                |
| 互相举报         | ACB 举报门店违规(强制摊派)/ 门店举报 ACB(假冒)→ 平台仲裁                                     |
| 客户归属         | 历史客户归原门店,平台不抢;新客户按 Lead 来源归属                                             |
| 跨门店转籍        | 可申请跨门店 / 跨国转籍;新接收方向旧方付一次性培训补偿金(平台撮合)                                     |
| 国家代理 / 门店替换  | **代理被替换** → ACB 身份不变、ACB↔门店挂靠不变,新代理接手治理;**门店被替换或解散** → ACB 走解绑流程,90 天冷静期 |
| 治理权(代理 / 门店) | 考核提案权、培训组织权、初级派单调度权;**无封号权 / 等级决定权 / 跨国派单权**                             |


### 9.2 国内 vs 海外的真正差异

挂靠关系全球统一,差异只在**门店之上是否多一层运营治理**:


| 层级                    | 国内           | 海外                |
| --------------------- | ------------ | ----------------- |
| Aqara 总部              | ✓ 直管门店       | ✓ 委托治理            |
| 国家代理 / Operator-Store | ✗ 不存在        | ✓ 治理本国所有门店与挂靠 ACB |
| Aqara Space 门店        | ✓(现有服务商升级品牌) | ✓(加盟筹备)           |
| ACB ↔ 门店挂靠规则          | 同一套(§9.1)    | 同一套(§9.1)         |
| Builder Console 代码    | 一套           | 一套(差异是配置项)        |


> 国家代理只是海外的运营叠加层,代理可被 Aqara 替换,门店和 ACB 身份均不变 — 学 Apple 切换 Country Distributor 时 AASP 无感。

### 9.3 为什么挂靠关系必须全球统一(设计原则)

1. **Builder Console 复用** — 一套代码跑全球,差异化通过配置项(数据中心 / 合规 / 支付通道)而非分支代码实现
2. **ACB 体验全球一致** — 注册 / 接单 / 钱包 / 提现 / 挂靠 / 解绑流程一样,降低培训和支持成本
3. **代理可换、门店可换、ACB 不动** — 这是 Aqara 全球化的根本保险,任何"代理拥有 ACB"的设计都会让代理变成诸侯
4. **国内服务商迁移友好** — 沿用现有挂靠制度,服务商升级到 Aqara Space 品牌时,关系模型不变,只是品牌和 AI 工具增强(详见 §13)

### 10.3 Operator-Store(门店即运营层)过渡模式

> **新机制**:为"无国家代理国"提供过渡运营总管能力,首个试点 = **意大利米兰**。

```
正常海外结构(韩国 / 迪拜):     Operator-Store 过渡结构(米兰):
┌────────────────┐               ┌──────────────────────────┐
│ 国家代理         │              │ Aqara Space 米兰门店       │
│ (运营/仓储/培训)  │             │ (复合身份)                  │
└──────┬─────────┘               │  • 门店职能: 服务交付        │
       │                         │  • Operator 职能(临时):     │
       ▼                         │    本国仓储 / 培训 / 初审      │
┌────────────────┐               └──────────┬────────────────┘
│ Aqara Space 门店 │                          │
└──────┬─────────┘                          ▼
       │                          ┌──────────────────┐
       ▼                          │ ACB 个人          │
┌────────────────┐                └──────────────────┘
│ ACB 个人         │
└────────────────┘
                                  退出: 国家代理建立后,
                                  Operator 职能剥离,
                                  门店退化为普通 Aqara Space
```

**4 个关键设计**:

1. **复合身份带过渡期标签**(`operator_role: temporary`),代理建立时自动剥离
2. **能力受限**:不能审批新门店(总部独家),不能改 ACB 等级,不能改价格底盘 — 沿用"五权留中"
3. **激励**:Operator 职能期间额外拿运营管理费(类似国家代理通道分润),退出时一次性 Operator 完成奖
4. **退出机制**:国家代理上任后,门店主可优先竞标继续做"国家代理"(若意愿 + 能力达标),或者纯化为门店

## 十三、国内服务商 → ACB 平滑迁移路径

> 这是**升级,不是替换**。让国内现有服务商觉得是"拿到新武器",不是"被收编"。

### 13.1 战略核心:4 件事拉动服务商加入

参考用户战略点名:**平台做好 + AI 赋能工具做好**。

1. **AI 工具赋能** — Builder Copilot / Persona Composer 专业版,服务商有了 BD 新武器
2. **Lead 派单流量** — Aqara Life 平台 C 端用户流量倾斜给挂靠 Aqara Space 门店
3. **分润透明化** — ACB 钱包看得见,每单 70-85% 直接结算,不再黑箱
4. **门店分成持续 5-10%** — Aqara Space 门店在所有 ACB 订单中拿分成,服务商不被绕过

### 13.2 实体迁移映射


| 现有服务商体系   | 新 ACB 体系                               |
| --------- | -------------------------------------- |
| 服务商主体(公司) | Aqara Space 门店主体                       |
| 服务商管理员账号  | 门店 Admin(对内管理 ACB,对总部负责)               |
| 服务商技术员工   | ACB(Installer / Designer / Service)子角色 |
| 按业绩返利(今天) | 等级返利(业绩数据平移为冷启动等级)+ AI 工具赋能            |
| 服务商客户关系   | 留在原门店,不被平台抢走                           |
| 服务商内部协作   | 门店自定义内部关系(雇佣 / 合伙 / 独立承包)              |


### 13.3 改造路径


| 路径         | 适用对象  | 门槛                                     |
| ---------- | ----- | -------------------------------------- |
| **平移注册**   | 现有服务商 | 极低 — 补 Aqara Academy Badge 即可,业绩数据自动迁移 |
| **新人完整认证** | 新 ACB | 完整 Academy 培训 + 考试 + Badge + 资质审查      |


### 13.4 迁移过渡期

- **双轨运行 6-12 个月**:老服务商体系与新 ACB 体系并行

- 12 个月后老体系完全下线

## 十五、落地原则

1. **用户永远直付 Aqara,不直付 ACB** — 防跑路硬约束,所有形态遵守
2. **货服分离是结构性护城河** — 主货官方仓直发用户,ACB 空手上门;假货 / 卷款风险归零
3. **Aqara Space 门店准入由总部独家审批** — 代理只有推荐 + 反对权,不能养亲兵
4. **ACB ↔ Aqara Space 门店挂靠制度全球统一** — ACB 身份归总部、关系归门店;Builder Console 一套代码跑国内外,差异化通过配置项实现
5. **国家代理五权留中、五事下放** — 认证 / 等级 / 抽成 / 封号 / 派单 / 价格 / 数据所有权 留中,仓储 / 培训 / 营销 / 客服 / 仲裁初审 下放
6. **国家代理 / Operator-Store 不得过手 ACB 分润** — Stripe Connect 平台直付,红线
7. **米兰走 Operator-Store 过渡模式** — 无代理国"门店即运营层",代理建立后剥离
8. **国内服务商迁移是升级不是替换** — 4 件事拉动:AI 工具 / Lead 流量 / 钱包透明 / 门店分成
9. **本地优先承诺不变** — 服务交付完成后 Studio 本地运行不依赖订阅持续
10. **跨境派单 MVP 禁用** — 数据 / 税务 / 资质合规风险高,V2 再开
11. **ACB 备件库 MVP 不开** — 全走官方仓直发,V2 再分层放开
12. **MVP 等级架构层支持 N 级动态配置** — 具体级数和分润比例交给运营规则,不写死代码
